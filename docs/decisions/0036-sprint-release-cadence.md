# ADR-0036 — Sprint + Release Cadence

## Status
Accepted

## Context
A sprint structure was needed to organise delivery from scaffold through Play submission.

## Decision
See `sprint-plan.md`. Sprint 0 = design + scaffold; Sprints 1–4 = MVP; Sprints 5–6 = hardening + Play submission.

## Consequences
All feature work is assigned to a sprint with a defined gate, and the sprint plan document is the authoritative source for scope per sprint.
