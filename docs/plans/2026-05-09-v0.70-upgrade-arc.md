### Key v7 architecture: Compute Unification
- `ComputeNode` = hardware/endpoint (GPU box, cloud, remote host)
- `LLM` = a named model on a compute node, any of 10 kinds: `ollama, openwebui, opencode, opencode-acp, opencode-prompt, claude-code, aider, goose, gemini, shell`
- `OpenWebUI` = an LLM (kind=openwebui) referencing an Ollama ComputeNode, NOT its own ComputeNode
- Session-backend LLMs (claude-code, opencode, aider, goose, gemini, shell) use ComputeNode as process host
- Legacy v6 per-backend cfg blocks are auto-migrated to LLM registry entries on first v7 daemon start
