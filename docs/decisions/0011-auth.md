# ADR-0011 — Auth

## Status
Accepted (superseded in part by ADR-0042)

## Context
An authentication scheme was needed that matched the parent server's static bearer token model without introducing pairing complexity in v1.

## Decision
Static bearer token per server profile (same as parent). Stored in EncryptedSharedPreferences backed by Android Keystore. No pairing flow, no biometric gate in v1. Domain abstracts a `Principal` for future multi-user support.

## Consequences
Auth is simple and consistent with the server, with biometric unlock deferred to Sprint 5 per ADR-0042.
